The OWASP (Open Worldwide Application Security Project) Top 10 for Large Language Model (LLM) Applications identifies and prioritizes the most critical security risks inherent in LLMs and the applications built upon them. This framework provides a structured methodology for analysing and mitigating Generative AI threats.

Here are five of the critical security risks identified within this framework, along with a brief explanation of each:

|   |   |   |
|---|---|---|
|OWASP Risk|Simple Explanation|Source(s)|
|**1. LLM01: Prompt Injection**|An attacker crafts a malicious prompt (e.g., a phrase like "Ignore all previous instructions...") that **overrides the AI's original safety rules or system instructions**, causing the model to perform unintended or unauthorized actions. This attack exploits the AI's struggle to separate trusted developer commands from untrusted user inputs.||
|**2. LLM02: Sensitive Information Disclosure**|The LLM **inadvertently reveals confidential data** that it has accessed at runtime or memorized from its training set. This can include sensitive data like Personally Identifiable Information (PII), proprietary source code, internal financial records, or trade secrets, leading to severe privacy and security violations.||
|**3. LLM04: Training Data Poisoning**|Adversaries intentionally **corrupt the AI's training data** by injecting malicious or manipulated samples. This attack embeds "sleeper" behaviors or systematic biases that remain dormant until activated by a specific trigger, leading to unpredictable failure or targeted misclassification once the model is deployed.||
|**4. LLM05: Insecure Output Handling**|This occurs when a downstream application blindly **trusts and fails to sanitize the content generated by the LLM**. An attacker can trick the AI into generating a malicious payload (like dangerous JavaScript or SQL commands) which then executes and attacks the connected application (e.g., leading to Cross-Site Scripting or XSS).||
|**5. LLM10: Model Theft**|An attacker systematically queries a proprietary model's public API to gather a sufficient amount of input and output data. This information is then used to **train a functional clone** (a surrogate model), effectively stealing the organization's intellectual property and competitive capabilities.||

These risks highlight that LLM security requires specialized practices because models interpret natural language probabilistically, rather than executing deterministic code, creating unique attack surfaces that conventional security tools often miss.
