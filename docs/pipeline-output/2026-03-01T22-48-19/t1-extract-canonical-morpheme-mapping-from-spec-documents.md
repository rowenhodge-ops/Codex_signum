## Part 1: Canonical Morphemes (from codex-signum-v3_0.md)

The Codex Signum specification defines **six morphemes** as the fundamental units of the grammar:

| Morpheme | Definition | Layer |
|----------|------------|-------|
| **Seed** | Atomic compute unit. Stateless, focused transformation. Models, functions, individual operations. | L1 (Atomic) |
| **Bloom** | Scoped composition with boundary. Composed patterns with defined scope and interface. | L2 (Composed) |
| **Resonator** | Coupling point. Relationship encoding, connection weight, dependency. | L3 (Relational) |
| **Grid** | Structural lattice. Spatial organization, clustering, topology. | L3 (Relational) |
| **Helix** | Temporal spiral. Evolution, learning trajectory, adaptation over time. | L4 (Temporal) |
| **Flow** | Directed movement. Data flow, feedback flow, cascade direction. | L4 (Temporal) |

### Key Spec Clarifications

1. **"Agent" is NOT a morpheme** — The spec explicitly rejects the "agent" framing:
   > "The 'agent' framing implies discrete identity, persistent self, autonomous decision-making, and bounded existence."

2. **"Pattern" is now "Bloom"** — Patterns are "coherent flows" that "compose functions with routing logic." In morpheme terms, this maps to Bloom (scoped composition with boundary).

3. **Models are substrate** — "Models are infrastructure, not participants." They are Seed morphemes when represented in the grammar.

4. **Functions are Seeds** — "Most operations are focused transformations — extract, classify, route, validate, generate. They don't need names. They need clarity of purpose." → seedType: "function"
